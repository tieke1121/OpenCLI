# 市场情报选品系统 - 产品需求文档 (PRD)

> **版本**: v1.0 MVP
> **更新时间**: 2026-04-16
> **状态**: 待开发

---

## 1. 产品愿景

### 1.1 一句话描述

帮助跨境电商卖家从"发现趋势"到"找到货源"的一站式 AI 驱动工具。

### 1.2 解决的问题

- 手动在多个平台切换收集数据，效率低
- 难以发现新兴趋势产品
- 货源匹配靠人工搜索，容易遗漏

### 1.3 核心价值

**AI 驱动的市场洞察 + 自动化货源匹配**

---

## 2. 目标用户

| 用户画像 | 特征 | 痛点 |
|----------|------|------|
| **小微卖家** | 个人/夫妻店，月销 $5k-50k | 选品靠感觉，缺乏数据支撑 |
| **创业团队** | 3-5人，选品为业务核心 | 调研耗时，选错品库存压力大 |
| **工厂型卖家** | 有工厂，想扩品类 | 不知道什么产品在亚马逊好卖 |

---

## 3. 用户使用场景

### 场景 A：日常选品调研

```
用户打开系统
    │
    ▼
输入品类关键词："收纳"
    │
    ▼
系统自动执行：
    1. 抓取 Amazon Best Sellers 榜单
    2. 抓取 Reddit 相关讨论
    3. AI 聚合分析趋势
    │
    ▼
展示：热门产品、讨论热度、竞争度
    │
    ▼
用户点击某个产品："可折叠收纳盒"
    │
    ▼
系统自动搜索 1688 货源
    │
    ▼
展示：供应商列表、价格、MOQ、利润估算
    │
    ▼
用户下载完整报告
```

### 场景 B：竞品监控

```
用户添加竞品监控任务
    │
    ▼
设置监控：
    - Amazon ASIN：B0XXXXX
    - 监控频率：每日
    │
    ▼
系统每日自动抓取：
    - 价格变化
    - 评论增长
    - 评分变化
    │
    ▼
生成竞品动态报告
```

## 4. 功能规格

### 4.1 数据采集模块

| 功能 | 描述 | 数据源 | 优先级 |
|------|------|--------|--------|
| Amazon 销售榜 | 抓取指定品类的 BS 榜单 | Amazon | P0 |
| Reddit 讨论 | 抓取指定关键词的热门帖子 | Reddit | P0 |
| 1688 货源 | 搜索商品、供应商、价格 | 1688 | P0 |
| 扩展预留 | 可接入 Twitter, TikTok, Instagram | (未来) | P2 |

**扩展机制**：适配器模式，新增数据源只需实现接口，无需改动核心逻辑

### 4.2 AI 分析模块

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 趋势聚合 | 从多源数据中发现共同趋势 | P0 |
| 产品关联 | 将社交热度与销售数据关联 | P0 |
| 竞争分析 | 评估市场竞争度 | P1 |
| 选品建议 | AI 推荐高潜力产品 | P0 |

### 4.3 货源匹配模块

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 1688 搜索 | 根据 AI 建议搜索货源 | P0 |
| 供应商排序 | 按价格/评分/MOQ 排序 | P0 |
| 利润计算 | (Amazon价 × 汇率 - 1688价) × 系数 - 运费 | P0 |
| 样品追踪 | 记录样品申请状态 | P2 |

### 4.4 报告生成模块

| 功能 | 描述 | 格式 | 优先级 |
|------|------|------|--------|
| 选品分析报告 | 趋势 + 货源 + 利润 | Markdown / PDF | P0 |
| 竞品监控报告 | 价格/评论/评分变化 | Markdown / PDF | P1 |
| 每日摘要 | 监控的竞品动态汇总 | Markdown / PDF | P2 |

### 4.5 监控调度模块

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 定时任务 | 按 Cron 表达式调度 | P1 |
| 手动触发 | 一键立即执行 | P0 |
| 执行日志 | 记录每次运行结果 | P1 |
| 失败重试 | 自动重试机制 | P1 |

## 7. 非功能需求

| 维度 | 指标 |
|------|------|
| **性能** | 一次完整分析 < 3 分钟（包含多源爬取） |
| **可用性** | 7x24 运行，SLA 99% |
| **扩展性** | 新增数据源 < 1 天接入 |
| **数据保留** | 报告保留 1 年，价格历史保留 2 年 |
| **安全** | API Key 加密存储，用户数据隔离 |
